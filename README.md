# Monica 个人关系管理系统 (PRM)

这是一个基于 React 和 Supabase 构建的个人关系管理系统（Personal Relationship Management），旨在帮助您记录和可视化与朋友、家人、同事等的人际关系网络，记录共同经历的人生时刻。

项目地址：[https://github.com/65316571/start-monica](https://github.com/65316571/start-monica)

## 🌟 核心功能

### 👥 人物管理
*   **全方位档案**：记录联系人的基本信息（姓名、性别、联系方式），以及身份关系、认识时间、现居城市（支持省市级联）、所属行业等深度信息。
*   **多维度筛选**：支持按姓名、性别、地区、行业、标签等多种条件组合筛选，快速定位目标人物。

### 🏷️ 标签系统
*   **自定义标签**：创建如 #同学、#同事、#家人 等个性化标签。
*   **可视化配色**：支持自定义每个标签的颜色，这些颜色将直接应用于关系图谱的连线中，直观区分关系类型。
*   **灵活管理**：随时编辑标签名称与颜色配置。

### 📅 事件记录
*   **互动追踪**：记录聚会、会议、聊天等互动事件。
*   **智能分组**：事件列表按“年-月”自动分组，清晰展示时间脉络。
*   **便捷录入**：在添加参与人时，支持通过标签筛选快速查找特定圈子的人。
*   **灵活排序**：支持按日期正序或倒序查看，支持按年份、月份筛选。

### ⏳ 人生浏览 (Life Timeline)
*   **多维度时间轴**：提供**十年、年、月、周**四种时间视图，全景式回顾人生轨迹。
*   **交互式浏览**：支持拖拽浏览和时间导航，直观展示每个时间点发生的关键事件。
*   **全貌与细节**：从宏观的人生阶段到微观的每周日常，随心切换视角。

### 🕸️ 人际关系图谱 (仪表盘)
*   **可视化网络**：基于 D3 力导向图动态展示您的人际网络结构。
*   **智能连接**：
    *   **显式关系**：基于共同参与的事件自动建立连接。
    *   **隐式关系**：拥有相同标签的人自动形成社群连接。
*   **交互式筛选**：
    *   **专注模式**：点击特定标签，图谱仅展示该标签下的社交网络，自动隐藏无关节点。
    *   **视觉偏好**：支持开启/关闭“彩色标签连线”。关闭时，多重关系将智能合并为一条简洁连线，降低视觉干扰。

### 💾 数据管理
*   支持数据的导入与导出（JSON 格式），保障数据主权与迁移便捷性。

## 🛠️ 技术栈

*   **前端框架**：React 18, Vite
*   **样式方案**：Tailwind CSS
*   **图标库**：Lucide React
*   **可视化**：React Force Graph (2D)
*   **日期处理**：date-fns
*   **后端/数据库**：Supabase (PostgreSQL)

## 🚀 快速开始

### 1. 克隆项目

```bash
git clone https://github.com/65316571/start-monica.git
cd start-monica
```

### 2. 安装依赖

进入前端目录并安装依赖：

```bash
cd web
npm install
```

### 3. 配置环境变量

在 `web` 目录下创建 `.env` 或 `.env.local` 文件，填入您的 Supabase 配置：

```properties
VITE_SUPABASE_URL=您的Supabase项目URL
VITE_SUPABASE_ANON_KEY=您的Supabase匿名Key
```

> **提示**：如果您使用本地 Supabase，URL 通常为 `http://127.0.0.1:54321`。

### 4. 启动开发服务器

```bash
npm run dev
```

启动后，访问 `http://localhost:5173` 即可开始使用。

### 5. 构建生产版本

```bash
npm run build
```
构建产物将输出到 `web/dist` 目录，可部署至任何静态网站托管服务。

## 🗄️ 数据库结构

项目依赖 Supabase (PostgreSQL)，核心表结构如下：

*   `people`: 存储人物档案（含地区、行业等扩展字段）。
*   `events`: 存储互动事件记录。
*   `tags`: 存储标签定义及其颜色配置。
*   `person_tags`: 人物与标签的关联表。
*   `event_participants`: 事件与参与人的关联表。
*   `relationships`: 存储人物间的显式关系强度。

数据库迁移文件位于 `supabase/migrations` 目录中，可直接在 Supabase SQL Editor 中执行以初始化数据库。
